# AI工程实践暑期实习培养计划（第一阶段）

## 一、培养目标

### 实习目标

本次实习并不以完成多少任务作为评价标准，而是希望帮助你完成从**学生**到**工程实践者**的第一次转变。

整个第一阶段重点培养以下四项能力：

* **自主学习能力** —— 学会独立查阅资料、分析问题、利用 AI 工具解决问题。
* **工程实践能力** —— 掌握 Git、Python、开发环境等工程基础，能够独立完成项目部署与开发。
* **知识管理能力** —— 建立属于自己的知识体系，形成长期积累和持续输出的习惯。
* **问题分析能力** —— 学会发现问题、分析问题，而不仅仅是运行代码或完成任务。

希望在整个过程中逐渐建立良好的工程习惯，而不仅仅是掌握某几项具体技术。

---

## 二、整体学习方式

整个阶段坚持以下学习流程：

```text
明确任务
    ↓
自主尝试
    ↓
使用 AI 辅助
    ↓
查阅官方文档
    ↓
查阅 GitHub / 技术资料
    ↓
整理理解
    ↓
形成知识沉淀
    ↓
Review
```

原则：

> **没有经过自己的思考，不直接询问导师。**

导师主要负责方向指导，不负责即时解决每一个具体问题。

---

# 三、整个暑期需要完成的五个任务

---

## Task 1 —— 工程基础能力建设

目标：

建立 AI 工程开发所需要的基础工程能力。

主要内容：

* Git / GitHub
* Markdown
* Python 环境
* IDE 使用
* 虚拟环境
* requirements
* GitHub 协作流程
* API Key 管理
* .gitignore
* .env

最终要求：

能够独立恢复一个 GitHub 项目并开始开发。

---

## Task 2 —— 建立个人知识库（Learning Notes）

目标：

建立长期维护的个人技术知识库。

要求：

* 使用 Markdown
* GitHub 管理
* 分类整理
* 长期维护

知识来源包括：

* 学习过程中整理
* 项目踩坑记录
* AI 对话总结
* 官方文档学习
* 行业资料阅读

重点不是记笔记，而是形成自己的知识体系。

---

### 输出要求

整个暑期至少完成若干篇能够公开分享的内容。

例如：

* Git 学习心得
* Python 踩坑记录
* Prompt 学习
* Agent 项目总结
* API 调试记录

可以发布：

* GitHub
* CSDN
* 掘金
* 小红书
* 公众号
* 知乎

不要求每天输出，而要求每一次输出都有一定价值。

---

## Task 3 —— AI Agent 项目实践

项目：

股票分析 Agent

目标：

通过真实项目学习现代 AI Agent 的开发流程。

主要内容：

* 环境部署
* GitHub 协作
* 项目理解
* Prompt
* Knowledge
* LLM
* Pipeline
* 工程结构

重点：

理解 Agent 为什么这样设计，而不仅仅是能够运行。

最终输出：

项目理解文档

---

## Task 4 —— 工业 AI 实践

项目：

音频异常检测

目标：

了解真实工业 AI 项目的研发流程。

包括：

第一阶段：环境部署

第二阶段：理解整个项目

第三阶段：阅读代码

第四阶段：实验

第五阶段：分析问题

重点不是调模型，而是理解：

* 为什么这样设计
* 为什么会误报
* 为什么需要数据
* 为什么模型不能直接上线

最终输出：

项目分析报告。
---
## Task 5 —— 行业认知

目标：

建立对 AI 产业链和相关技术生态的整体认识，理解项目所处的位置和应用场景。

学习内容：

* AI 行业与产业地图
* 大模型与 AI Agent 的发展趋势
* 工业 AI 的典型应用
* 国产算力、GPU、智算中心等基础设施
* 存储、数据、模型之间的关系

要求：

* 阅读导师提供的资料。
* 不要求深入掌握细节，但需要能够说清楚各个技术的大致作用及相互关系。
* 每完成一个主题，可整理一页自己的理解或思维导图，作为知识库的一部分。

---

# 四、日常工作要求

每天：

* 制定当天计划
* 记录学习过程
* 总结当天收获

重点记录：

* 今天学到了什么
* 遇到了什么问题
* 怎样解决
* 明天准备做什么

全部采用 Markdown。

---

# 五、遇到问题时

按照下面顺序：

1. 阅读错误信息

2. 自己分析

3. AI

4. 官方文档

5. GitHub Issues

6. 技术社区

7. 导师

向导师提问时，需要说明：

* 问题

* 已尝试方案

* 查阅资料

* 自己判断
